# CLI reference

```
docs-portal [build] [options]
docs-portal init [options]
```

Running `docs-portal` with no subcommand is equivalent to `docs-portal build`.

## `build`

Generates the HTML pages, the portal index and an `llms.txt` context index.

| Option | Default | Description |
| --- | --- | --- |
| `--root PATH` | current directory | Directory to scan. |
| `--out NAME` | `DOCUMENTATION.html` | Filename of the portal index, written in the root. Use `index.html` for web hosting. |
| `--config PATH` | `docs-portal.toml` in the root | Configuration file. |
| `--no-timestamp` | off | Omit the build time from output for reproducible, byte-identical builds and clean diffs. |
| `--quiet` | off | Suppress the build report. |

Example:

```bash
docs-portal build --root ./site --out index.html --no-timestamp
```

### Build report

Unless `--quiet` is given, the build prints a summary, for example:

```
Build summary
  Root:           /path/to/site
  Config:         default (no docs-portal.toml)
  Markdown (12):   1 created · 2 updated · 9 unchanged · 0 skipped
  Manual docs:    0 checked · 0 updated · 0 unchanged
  Existing HTML:  4 linked
  Portal:         index.html (updated)
  LLM context:    llms.txt (updated)
```

- **created / updated / unchanged** — pages created, changed, or left untouched.
- **skipped** — `.md` files whose `.html` exists but was *not* generated by the
  tool (no marker); these are never overwritten.
- **Existing HTML** — hand-written `.html` files indexed into the portal.
- **LLM context** — generated `llms.txt` index. Existing manual files without
  the generated marker are skipped instead of overwritten.

## `init`

Generates a `docs-portal.toml` by detecting chapters from the top-level folders
that contain documents.

| Option | Default | Description |
| --- | --- | --- |
| `--root PATH` | current directory | Directory to inspect. |
| `--config PATH` | `docs-portal.toml` in the root | Where to write the config. |
| `--force` | off | Overwrite an existing config file. |

Example:

```bash
docs-portal init --root ./site
```

Folders without any `.md`/`.html` are ignored; hidden folders and build noise
(`node_modules`, `.git`, `dist`, `build`, …) are always skipped.

## Exit behavior

- `docs-portal --help` and `docs-portal <command> --help` print usage.
- `init` prints a message and makes no changes if the config already exists and
  `--force` was not given.
